# KODA.md — Инструкции для работы с проектом Wishlist App

## Обзор проекта

**Wishlist App** — это полноценное веб-приложение для создания и управления вишлистами с функцией группового дарения подарков. Проект представляет собой социальную платформу, где пользователи могут создавать списки желаний для различных событий (дни рождения, свадьбы, праздники), делиться ими с друзьями и получать подарки анонимно.

Проект построен по архитектуре **Frontend + Backend** и использует современные технологии для обеспечения красивого пользовательского интерфейса и real-time функциональности.

## Технологический стек

### Frontend (директория `app/`)

- **React 19** — основной фреймворк для построения пользовательского интерфейса
- **TypeScript** — типизация для повышения надёжности кода
- **Vite 7** — инструмент сборки и сервер разработки
- **Tailwind CSS 3.4** — утилитарный CSS-фреймворк для стилизации
- **Framer Motion 12** — библиотека для анимаций и визуальных эффектов
- **shadcn/ui** — коллекция готовых UI-компонентов на основе Radix UI (40+ компонентов)
- **Axios** — HTTP-клиент для работы с API
- **React Router DOM 7** — маршрутизация на клиенте
- **React Hook Form** — управление формами
- **Zod** — валидация данных
- **Lucide React** — иконки

### Backend (директория `backend/`)

- **FastAPI** — веб-фреймворк на Python
- **Uvicorn** — ASGI-сервер
- **Pydantic** — валидация данных и модели
- **Python-Jose** — работа с JWT-токенами
- **Passlib** — хеширование паролей

## Сборка и запуск проекта

### Предварительные требования

- **Python 3.8+** — для запуска backend
- **Node.js 18+** — для запуска frontend
- **npm** — менеджер пакетов (входит в состав Node.js)

### Запуск Backend

```bash
cd backend
python3 -m venv venv
# На Windows: venv\Scripts\activate
source venv/bin/activate
pip install -r requirements.txt
python3 main.py
```

Backend будет доступен по адресу: `http://localhost:8000`

### Запуск Frontend

```bash
cd app
npm install
npm run dev
```

Frontend будет доступен по адресу: `http://localhost:5173`

### Доступные npm-скрипты

| Команда | Описание |
|---------|----------|
| `npm run dev` | Запуск сервера разработки |
| `npm run build` | Сборка проекта (проверка типов + сборка) |
| `npm run lint` | Запуск линтера ESLint |
| `npm run preview` | Предпросмотр собранного проекта |

## Архитектура и паттерны

### Frontend

- **Контекст AuthContext** — управление состоянием аутентификации (src/contexts/AuthContext.tsx)
- **API-клиент** — централизованные HTTP-запросы с интерцепторами для токенов (src/lib/api.ts)
- **Приватные маршруты** — защищённые страницы требуют авторизации
- **TypeScript-типы** — единые интерфейсы для данных (src/types/index.ts)

### Backend

- **In-memory хранилище** — данные хранятся в памяти (не персистентное)
- **WebSocket-менеджер** — класс ConnectionManager для управления подключениями
- **JWT-аутентификация** — токены с временем жизни 7 дней
- **Зависимости через FastAPI Depends** — внедрение зависимостей для защищённых эндпоинтов

## Стиль кодирования

### Frontend (TypeScript/React)

- **Именование:** camelCase для переменных и функций, PascalCase для компонентов и типов
- **Импорты:** используется абсолютный импорт через `@/` (настроено в tsconfig)
- **Компоненты:** функциональные компоненты с хуками
- **Типизация:** все данные имеют явные TypeScript-интерфейсы
- **CSS:** Tailwind CSS с утилитарными классами

### Backend (Python)

- **Стиль:** соответствует стандартам FastAPI
- **Модели:** Pydantic BaseModel для валидации
- **Асинхронность:** async/await для всех эндпоинтов
- **Типизация:** аннотации типов Python

## Основные функции

1. **Создание вишлистов** — пользователь может создавать вишлисты для различных событий с названием, описанием и типом события
2. **Публичные ссылки** — каждый вишлист имеет уникальный slug для публичного доступа
3. **Анонимные резервирования** — друзья могут резервировать подарки, не раскрывая свою личность владельцу
4. **Групповые подарки** — несколько друзей могут вносить деньги на дорогие подарки
5. **Real-time обновления** — WebSocket обеспечивает мгновенное отображение изменений
6. **Визуальные эффекты** — анимации, конфетти, частицы для праздничного настроения

## Заметки по разработке

- Backend использует **in-memory хранилище**, поэтому данные теряются при перезапуске сервера
- Для production необходимо добавить базу данных (SQLite, PostgreSQL)
- SECRET_KEY в коде должен быть заменён на безопасное значение
- CORS настроен для всех источников (`allow_origins=["*"]`) — для production следует ограничить
- Фронтенд использует React 19 с современными возможностями
